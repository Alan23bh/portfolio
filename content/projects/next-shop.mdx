### This project focuses primarily on frontend architecture, state management,and building reusable UI components rather than backend integrations.

---

## Key Features

• Fully responsive product catalog using Material UI  
• Global cart state using React Context + useReducer  
• Add, remove, increment, and decrement cart items  
• Automatic total price calculation  
• Snackbar feedback for user actions  
• Local product dataset for reliable production deployment

---

## Overview

This project is a frontend-focused e-commerce interface designed to simulate
real-world shopping interactions such as browsing products and managing a cart.

The goal was to build a structured and maintainable UI while practicing global
state management patterns commonly used in React applications.

---

## Problem

Managing shared state across multiple components can quickly become complex in
React applications, especially when handling interactions like adding and
removing items from a cart.

I wanted to explore a predictable and scalable approach to handling global
state without relying on external libraries.

---

## Solution

I implemented a global cart system using React Context combined with
useReducer to manage state transitions in a clear and structured way.

The UI was built using Material UI components to maintain consistency, while
the data layer was simplified by using a local dataset to avoid dependency on
external APIs during deployment.

---

## What I built

Key elements of the project include:

• Product grid layout with responsive design  
• Product cards with actions (view, add to cart)  
• Global cart system with reducer-based state management  
• Quantity controls and item removal  
• Real-time total calculation  
• Snackbar notifications for user feedback  
• Fully responsive layout across devices

---

## Technical Highlights

• Next.js App Router with TypeScript  
• React Context + useReducer for global state  
• Material UI for component-based design system  
• Local dataset for stable deployment  
• Component-based architecture for scalability

---

## Lessons learned

This project reinforced how important predictable state management is when
building interactive applications. Using useReducer helped me better
understand how to structure state updates in a controlled and scalable way.

I also learned how separating data from UI logic and avoiding unnecessary API
dependencies can make applications more reliable and easier to deploy.
